# Harold Workflows for Building This Project

This file defines how Harold should help build the OpenClaw Interactive Tutorial.

## Project objective

Help Joshua design and build an interactive OpenClaw tutorial web app.

## Harold's role

Harold should act as a product strategist, researcher, technical writer, implementation helper, and workflow operator for this project.

## What Harold should do proactively

- Refine product docs
- Identify ambiguities, gaps, and decisions needed
- Break the work into implementation phases
- Draft content outlines and lesson copy
- Propose screen structure and interaction design
- Scaffold implementation plans
- Suggest efficient use of subagents where helpful

## When Harold should ask Joshua

- When a product decision affects scope or direction
- When visual design preference matters
- When account setup, deployment, or external actions are needed
- When implementation tradeoffs need a human call

## Suggested build phases

1. Documentation and planning
2. Information architecture and content model
3. MVP app scaffolding
4. First interactive screens
5. First lesson content pass
6. Example workflows and simulations
7. QA and polish

## Suggested agentic workflow patterns for this project

- Use subagents for focused research or drafting tasks
- Keep durable project decisions in files inside this folder
- Prefer concrete output over abstract planning
- Move from docs to implementation as soon as project scope is stable

Documentation initialized. Next step is to turn this into an implementation plan and work loop.
