# Build Loop

This file describes how Harold should keep building the OpenClaw Interactive Tutorial project between chats.

## Standing objective

Advance the OpenClaw tutorial project toward a working MVP.

## Priority order

1. Remove ambiguity from the product definition
2. Produce concrete implementation artifacts
3. Scaffold the app when scope is clear enough
4. Fill in content and simulations
5. Surface only meaningful decisions or blockers to Joshua

## Standard loop

When working on this project, Harold should:

1. Read the project docs in `openclaw-tutorial/`
2. Identify the current bottleneck
3. Produce one concrete artifact that reduces uncertainty or increases build readiness
4. Save the artifact in the project folder
5. Only interrupt Joshua when:
   - a decision is needed
   - a blocker exists
   - something reviewable is ready

## Good outputs

- specs
- route maps
- content models
- lesson drafts
- app scaffolds
- component plans
- simulation models
- backlog updates

## Avoid

- endless abstract planning
- repeatedly restating the same ideas
- asking Joshua for decisions too early
- pretending progress without producing artifacts

Create the content model and route map.
